## DeepSeek-V3 在 DeepSeek-V2 的基础上，从架构、训练效率、后训练和部署等多个方面进行了显著改进 

------

### 一、架构创新

1. **辅助损失自由的负载均衡策略（Auxiliary-Loss-Free Load Balancing）**
   - V2 依赖辅助损失来鼓励专家负载均衡，但过大的辅助损失会损害模型性能。
   - V3 引入动态偏置项调整专家负载，避免使用辅助损失，从而减少对模型性能的负面影响，并实现更好的专家 specialization。
2. **多令牌预测（Multi-Token Prediction, MTP）**
   - V2 仅预测下一个 token。
   - V3 同时预测多个未来 token（实验中使用深度 D=1，即多预测一个 token），提升训练信号密度和数据效率，同时可用于推测解码以加速推理。
3. **共享专家与路由专家的结构延续**
   - V3 继续采用 DeepSeekMoE 架构（细粒度专家 + 共享专家），但 V3 使用 sigmoid 计算亲和度并对选定专家进行归一化，V2 使用 softmax。

------

### 二、训练效率与基础设施升级

1. **FP8 混合精度训练（首次在超大规模模型上验证）**
   - V2 使用 BF16 或 FP32。
   - V3 引入细粒度量化（tile-wise / block-wise）和高精度累加策略，支持 FP8 训练，显著降低显存和通信开销，提升训练速度。
2. **DualPipe 流水线并行算法**
   - 比 V2 使用的 1F1B 或 ZB1P 更高效，减少流水线气泡，实现计算与通信的完全重叠，尤其适合跨节点 MoE 训练。
3. **高效的跨节点 All-to-All 通信**
   - 利用 NVLink 和 IB 带宽，结合 warp specialization 和动态调度，实现低开销、高吞吐的专家并行通信。
4. **内存优化**
   - 重计算 RMSNorm 和 MLA 上投影，EMA 参数存储在 CPU，MTP 模块与主模型共享 embedding 和输出头，显著降低激活内存。

------

### 三、训练数据与扩展能力

1. **更大规模、更多样的预训练数据**
   - V2 训练数据未明确规模，V3 使用 14.8T tokens，涵盖更多数学、编程和多语言样本。
2. **更长上下文扩展**
   - V3 通过两阶段 YaRN 扩展，从 4K → 32K → 128K，并在 NIAH 测试中表现稳健。
3. **Fill-in-the-Middle（FIM）策略**
   - 从 DeepSeekCoder-V2 引入，不损害下一 token 预测能力，提升模型对上下文的理解能力。

------

### 四、后训练与推理能力增强

1. **从 DeepSeek-R1 蒸馏推理能力**
   - V3 在 SFT 和 RL 阶段引入 R1 的长链推理（Long-CoT）数据，显著提升数学、代码等推理任务的性能，同时控制输出长度。
2. **自奖励机制（Self-Rewarding）**
   - 使用 V3 自身作为生成式奖励模型，结合投票机制提供反馈，提升对齐效果。
3. **推理部署优化**
   - 预填充阶段使用 TP4 + SP + DP8 + EP32，解码阶段使用 EP320 + 冗余专家策略，支持动态负载均衡和高吞吐低延迟推理。

- **更高效的架构**：无辅助损失负载均衡 + 多 token 预测。
- **更强的训练效率**：FP8 训练 + DualPipe + 跨节点通信优化。
- **更强的推理能力**：从 R1 蒸馏 + 自奖励 + 长上下文支持。
- **更高的性价比**：671B 参数、37B 激活，训练成本仅 5.576M 美元，性能接近 GPT-4o 和 Claude-3.5-Sonnet。
